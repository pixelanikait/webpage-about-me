My Projects

Music player backend

A command line C implementation of a music player that handles queues, playlists,albums,etc. Built to understand data structures like linked-list and their uses in real applications.

One tradeoff I made was sacrificing the ability to handle extremely large music libraries so I could use simpler data structures.

Doing it again, I would probably allow it to work more efficiently for bigger libraries and maybe be able to handle multiple users at once.

Twixt

A command line implementation of the game twixt in C. It auto creates links between pegs and checks for link intercepts. I built it to get a understanding of graphs and depth searches.

One tradeoff I made was sacrificing efficiency for a more robust link check system along with checking for a win on every move.

Doing it again, I would implement a more efficient win-check and linking algorithm. I would also like to change the current interface into a pop-out interface so users could point and click pegs instead of inputting coordinates as sometimes coordinates can be misread.

Spam message checker

A python application that uses command line to take input and display output. Taking a message it uses TF-IDF vectorization along with a Multinomial Naive-Bayes algorithm to check if a message is spam. I built it to gain a better understanding of text vectorization and models.

One tradeoff I made was the necessity to retrain the model every time it predicted an input wrong. It was also extremely hard to direct the model to not drop certain words that would be a good indicator if a message was spam.

I would like to further improve the project by maybe testing a few more models along with maybe linking it to my email or whatsapp so it has some practical usage. I would also like to increase the size of the data-set to include more realistic examples.
